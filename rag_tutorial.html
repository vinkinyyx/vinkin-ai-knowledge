PHASE 0

🎒准备阶段：5 件事 + 心理建设

在敲代码之前，先把"硬件 + 心理"都准备好。准备不充分，敲 100 行代码也跑不通。

心理准备 1：RAG 没那么玄乎

💡 RAG 一句话理解

RAG = 检索 + 生成。你给 LLM 一份"开卷考试资料"（你公司的文档），LLM 根据资料回答问题。
没有 RAG = LLM 凭记忆回答（容易胡编）。
有 RAG = LLM 带着"参考资料"回答（有据可查）。

心理准备 2：你会遇到的 3 类挫折

阶段常见问题真实原因破解方法
环境搭建pip install 失败Python 版本 / 网络用 conda 或换源
文档处理PDF 读出来是乱码扫描版 / 加密换工具（pymupdf / marker）
检索效果回答不相关切片太大 / 关键词没命中调小切片 + 加 BM25

PHASE 3

🧹文档清洗：5 步规范化（1 小时）

原始文档有很多"脏数据"——多余空行、页眉页脚、乱码。本阶段把它们清理干净，让 RAG 用得更顺。

11
创建清洗脚本

在项目根目录新建 step1_clean.py：

import re
from pathlib import Path
from langchain_community.document_loaders import (
    PyPDFLoader,
    Docx2txtLoader,
    TextLoader,
)
from langchain_core.documents import Document

RAW_DIR = Path("raw_docs")
CLEAN_DIR = Path("clean_docs")
CLEAN_DIR.mkdir(exist_ok=True)

def clean_text(text: str) -> str:
    """5 步清洗：去空行/去页眉页脚/去乱码/合并空格/去URL"""
    # 1. 合并连续空行 → 1 个
    text = re.sub(r'\n\s*\n', '\n\n', text)
    # 2. 去行首尾空格
    text = '\n'.join(line.strip() for line in text.split('\n'))
    # 3. 合并连续空格 → 1 个
    text = re.sub(r' +', ' ', text)
    # 4. 去 URL
    text = re.sub(r'https?://\S+', '', text)
    # 5. 去 HTML 标签（如果有）
    text = re.sub(r'<[^>]+>', '', text)
    return text.strip()

加载不同格式文档

接上面的脚本，添加加载函数：

def load_document(file_path: Path) -> str:
    """根据后缀用不同的 loader"""
    suffix = file_path.suffix.lower()
    if suffix == ".pdf":
        loader = PyPDFLoader(str(file_path))
    elif suffix == ".docx":
        loader = Docx2txtLoader(str(file_path))
    elif suffix in [".md", ".txt"]:
        loader = TextLoader(str(file_path), encoding="utf-8")
    else:
        print(f"⚠️  不支持格式: {file_path.name}")
        return ""
    docs = loader.load()
    return "\n\n".join([d.page_content for d in docs])
13
遍历所有文档 + 保存
def main():
    files = list(RAW_DIR.glob("**/*"))
    files = [f for f in files if f.is_file()]
    print(f"📁 找到 {len(files)} 份文档\n")

    for f in files:
        print(f"⏳ 正在处理: {f.name}")
        raw_text = load_document(f)
        if not raw_text:
            continue
        clean_text = clean_text(raw_text)
        # 存为 .txt（最通用）
        out_path = CLEAN_DIR / (f.stem + ".txt")
        out_path.write_text(clean_text, encoding="utf-8")
        print(f"  ✅ {f.name} → {out_path.name} ({len(clean_text)} 字)\n")

    print(f"🎉 全部清洗完成！输出在 {CLEAN_DIR}/")

if __name__ == "__main__":
    main()

抽样检查清洗质量

打开 clean_docs/ 里的任意一个 txt 文件，人眼检查：

  • 没有连续 3+ 个空行
  • 没有页眉页脚残留（如"第 X 页 共 Y 页"）
  • 没有乱码（如 ????? 或 锟斤拷）
  • 段落结构清晰（标题和正文能区分）
  • 专业术语完整（如"GMP"没有被改成"G M P"）
❌ 常见清洗错误

如果发现大量问题，回到 step 12，在 clean_text() 里加更多清洗规则。

  • 表格变成乱码？用 marker-pdf 替代 pypdf（更适合表格）
  • 中英文标点混乱？加 text = re.sub(r'[,;]', '，', text)
  • 公式/图表丢失？这是 RAG 通病，跳过公式（用 GPT-4 Vision 单独处理）

PHASE 4

✂️文档切片：3 大策略（1 小时）

LLM 有"上下文窗口限制"（DeepSeek 是 32K tokens ≈ 5 万字）。你的文档可能有 100 万字——必须切成"小块"。
切片的好坏直接决定 RAG 的回答质量。

16
理解 3 大切片策略
📏 固定长度切片

每 N 个字切一段。最简单，但可能切断完整句子。

📄 按段落/句子切

按换行符或句号切。保留语义，但长短不一。

🧠 语义切片（推荐）

按"意思变化"切。最智能，但需要 Embedding。

17
切片策略 A：按段落切（推荐新手）

新建 step2_split.py：

from pathlib import Path
from langchain_text_splitters import RecursiveCharacterTextSplitter

CLEAN_DIR = Path("clean_docs")
CHUNKS_FILE = Path("chunks.jsonl")

# 配置切片参数
# chunk_size: 每段最大字符数
# chunk_overlap: 段与段之间重叠多少字（保持上下文连贯）
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,       # 500 字/段（中文最佳实践）
    chunk_overlap=50,      # 重叠 50 字（防止切断）
    separators=["\n\n", "\n", "。", "，", " ", ""],  # 优先按段落切
)

def split_documents():
    print("✂️  开始切片...\n")
    all_chunks = []
    files = list(CLEAN_DIR.glob("*.txt"))

    for f in files:
        text = f.read_text(encoding="utf-8")
        chunks = splitter.split_text(text)
        for i, chunk in enumerate(chunks):
            all_chunks.append({
                "id": f"{f.stem}_{i}",
                "source": f.name,
                "content": chunk,
            })
        print(f"  ✅ {f.name}: {len(chunks)} 段")

    import json
    with open(CHUNKS_FILE, "w", encoding="utf-8") as fp:
        for c in all_chunks:
            fp.write(json.dumps(c, ensure_ascii=False) + "\n")

    print(f"\n🎉 共 {len(all_chunks)} 段，存到 {CHUNKS_FILE}")

if __name__ == "__main__":
    split_documents()

PHASE 5

🧬Embedding：让机器"理解"文字（30 分钟）

Embedding = 把一段文字变成一组数字（向量）。意思相近的文字，向量也相近。这是 RAG 检索的"魔法"。

20
理解 Embedding 是什么
💡 1 句话理解

Embedding = 把每个字/词/句变成一个高维数字数组（如 1536 维）。

  • "我喜欢吃苹果" → [0.12, -0.34, 0.78, ...]（1536 个数字）
  • "我爱水果" → [0.15, -0.32, 0.81, ...]（非常接近！）
  • "今天天气不错" → [-0.55, 0.22, -0.13, ...]（差很远）

"距离近 = 意思像"——这就是 RAG 检索的数学基础。

新建 step3_embed.py：

import json
import os
import time
from pathlib import Path
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"],
    base_url=os.environ["OPENAI_API_BASE"],
)

CHUNKS_FILE = Path("chunks.jsonl")
EMBEDDINGS_FILE = Path("embeddings.jsonl")

def load_chunks():
    chunks = []
    with open(CHUNKS_FILE, "r", encoding="utf-8") as f:
        for line in f:
            chunks.append(json.loads(line))
    return chunks

def embed_batch(texts, batch_size=50):
    """批量调用 Embedding API（一次最多 50 条）"""
    all_vectors = []
    for i in range(0, len(texts), batch_size):
        batch = texts[i:i+batch_size]
        print(f"  ⏳ 处理 {i+1}-{i+len(batch)}...")
        resp = client.embeddings.create(
            model="text-embedding-3-small",
            input=batch,
        )
        all_vectors.extend([v.embedding for v in resp.data])
        time.sleep(0.5)  # 避免频率限制
    return all_vectors

def main():
    chunks = load_chunks()
    print(f"📊 共 {len(chunks)} 段，开始 Embedding...\n")

    texts = [c["content"] for c in chunks]
    vectors = embed_batch(texts)

    # 存为 jsonl（每行一段：id + source + content + vector）
    with open(EMBEDDINGS_FILE, "w", encoding="utf-8") as f:
        for chunk, vec in zip(chunks, vectors):
            f.write(json.dumps({
                "id": chunk["id"],
                "source": chunk["source"],
                "content": chunk["content"],
                "vector": vec,
            }, ensure_ascii=False) + "\n")

    print(f"\n🎉 Embedding 完成！存到 {EMBEDDINGS_FILE}")
    print(f"📦 文件大小: {EMBEDDINGS_FILE.stat().st_size / 1024 / 1024:.1f} MB")

if __name__ == "__main__":
    main()

import json
import numpy as np
import faiss
from pathlib import Path

EMBEDDINGS_FILE = Path("embeddings.jsonl")
INDEX_FILE = Path("faiss.index")
META_FILE = Path("metadata.jsonl")

def load_embeddings():
    chunks, vectors = [], []
    with open(EMBEDDINGS_FILE, "r", encoding="utf-8") as f:
        for line in f:
            d = json.loads(line)
            chunks.append({"id": d["id"], "source": d["source"], "content": d["content"]})
            vectors.append(d["vector"])
    return chunks, np.array(vectors, dtype="float32")

def main():
    chunks, vectors = load_embeddings()
    print(f"📊 加载 {len(chunks)} 段，向量维度 {vectors.shape[1]}")

    # 建索引（用最简单的 IndexFlatL2）
    dim = vectors.shape[1]
    index = faiss.IndexFlatL2(dim)
    index.add(vectors)

    print(f"✅ 索引建立，共 {index.ntotal} 个向量")

    # 存索引 + 元数据
    faiss.write_index(index, str(INDEX_FILE))
    with open(META_FILE, "w", encoding="utf-8") as f:
        for c in chunks:
            f.write(json.dumps(c, ensure_ascii=False) + "\n")

    print(f"💾 索引: {INDEX_FILE} ({INDEX_FILE.stat().st_size/1024/1024:.1f} MB)")
    print(f"💾 元数据: {META_FILE}")

if __name__ == "__main__":
    main()

理解 RAG 流程
💡 RAG 三步走
  1. 用户提问 → 拿问题
  2. 检索资料 → 用问题去向量库找最相关的 3-5 段
  3. LLM 生成 → 把"问题 + 资料"一起给 LLM，让它基于资料回答

核心 Prompt："请基于以下资料回答问题。如果资料里没有，请说'我不知道'。"

写 RAG 主程序

新建 rag.py（这是核心）：

import json
import os
import numpy as np
import faiss
from pathlib import Path
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"],
    base_url=os.environ["OPENAI_API_BASE"],
)

# 加载索引
index = faiss.read_index("faiss.index")
chunks = []
with open("metadata.jsonl", encoding="utf-8") as f:
    for line in f:
        chunks.append(json.loads(line))

def embed_query(query: str):
    resp = client.embeddings.create(
        model="text-embedding-3-small",
        input=query,
    )
    return np.array([resp.data[0].embedding], dtype="float32")

def retrieve(query: str, top_k: int = 3):
    q_vec = embed_query(query)
    distances, indices = index.search(q_vec, top_k)
    results = []
    for i, idx in enumerate(indices[0]):
        results.append({
            "rank": i+1,
            "distance": float(distances[0][i]),
            "content": chunks[idx]["content"],
            "source": chunks[idx]["source"],
        })
    return results

# 核心 Prompt 模板（这是 RAG 的灵魂）
RAG_PROMPT = """你是企业内部知识库助手。请严格基于【参考资料】回答用户问题。

【参考资料】
{context}

【用户问题】
{question}

【回答要求】
1. 只使用参考资料中的信息回答
2. 如果参考资料没有相关信息，请明确说"我在知识库中没有找到相关信息"
3. 回答末尾标注信息来源（如"来源：产品手册.pdf 第3章"）
4. 简洁清晰，3-5 句话
"""

def rag_query(question: str, top_k: int = 3):
    # 1. 检索
    refs = retrieve(question, top_k)

    # 2. 拼装 context
    context = "\n\n---\n\n".join(
        f"[来源 {r['rank']}: {r['source']}]\n{r['content']}"
        for r in refs
    )

    # 3. 调 LLM
    prompt = RAG_PROMPT.format(context=context, question=question)
    resp = client.chat.completions.create(
        model=os.environ.get("OPENAI_MODEL", "deepseek-chat"),
        messages=[{"role": "user", "content": prompt}],
        temperature=0.3,  # 低温度 = 更准
    )
    answer = resp.choices[0].message.content

    return {
        "answer": answer,
        "references": refs,
    }

if __name__ == "__main__":
    print("🤖 RAG 知识库助手已启动！\n")
    while True:
        q = input("\n💬 你的问题 (q 退出): ").strip()
        if q.lower() in ("q", "quit", "exit", ""):
            break
        result = rag_query(q)
        print(f"\n📖 回答:\n{result['answer']}")
        print(f"\n📚 参考资料 (前 3 段):")
        for r in result["references"]:
            print(f"  {r['rank']}. {r['source']} (距离: {r['distance']:.3f})")

BM25 是传统关键词检索，两者混合效果最好。

FAQ

❓10 个最常见问题

Q1: pip install 一直失败？

3 个解法：

  1. 换源：pip install -i https://pypi.tuna.tsinghua.edu.cn/simple xxx
  2. 用 conda：conda install xxx
  3. 升级 pip：python -m pip install --upgrade pip

Q2: 怎么知道 RAG 的质量？

写 10-20 道真实业务问题，让另一个人看回答打分：

  • 准确性（1-5）：回答是否正确
  • 有据可查（1-5）：是否引用了资料
  • 用户体验（1-5）：是否通顺

目标：3 项平均 ≥ 4 分。

Q3: 我的资料是图片 / 扫描版 PDF 怎么办？

2 个工具：

  1. marker-pdf（推荐）：pip install marker-pdf
  2. GPT-4 Vision：直接用 GPT-4 看图提取文字

Q4: 回答中文不准确？

3 个方向：

  1. 换中文专用 Embedding：BGE 中文 / 智谱 embedding
  2. 加 BM25 混合检索（中文场景必备）
  3. Prompt 里强调"用中文回答"

Q5: 怎么让回答更准？

按影响力排序：

  1. 清洗文档（去噪、去重）
  2. 调切片参数（chunk_size 300-800 都试试）
  3. 加 BM25 混合检索
  4. 加 ReRank 精排
  5. 优化 Prompt（让 LLM 严格基于资料）

Q6: API Key 怎么保密？

用 .env 文件 + python-dotenv 读取（上面步骤已经做）。

Q7: RAG 和微调（Fine-tuning）怎么选？

维度RAG微调
适合需要查具体资料需要改 LLM 行为 / 风格
成本低（每次查询几厘）高（训练几千元起）
数据更新实时（重新建索引）需重新训练
新手友好⭐⭐⭐⭐⭐⭐⭐

建议：99% 场景先用 RAG，不行再考虑微调。

Q8: 资料会泄露吗？

看 LLM 服务的隐私政策。

  • DeepSeek / OpenAI：默认不会用你的数据训练（opt-in 才用）
  • 敏感数据（医疗/法律/财务）：用本地模型（Ollama + Qwen）

Q9: 一次能处理多少资料？

取决于 Embedding API 的限制：

  • OpenAI text-embedding-3-small：一次 2048 条，总长度无上限
  • DeepSeek：一次 2048 条

本教程的 embed_batch() 函数已经按 50 条/批处理，1 万段 ≈ 5 分钟。

Q10: RAG 跟 LangChain / LlamaIndex 什么关系？

LangChain / LlamaIndex 是框架，帮你快速搭 RAG；本教程是从零手写。

建议：

  1. 先用手写（本教程）—— 理解每个步骤
  2. 再用框架加速（LangChain）—— 5 行代码搞定
