坑 01 · 数据层

PDF 暴力 + 固定切分

😰 现象 1：PDF 读取
多栏排版混在一起、表格变乱码、页眉页脚污染正文
😰 现象 2：固定切分
split_text(500) 死板切，一句话拦腰断，"他"找不到指代对象
💥 合并后果
检索片段语义不完整，模型"张冠李戴"，准确率跌到 60% 以下
5 / 11
✅ 坑 01 解决方案

版面分析 + NLP 动态切分

✓
复杂表格：让大模型转 Markdown 或写摘要
✓
图片/流程图：GPT-4o 多模态生成文字描述
✓
扫描件：PaddleOCR → 版面分析还原阅读顺序
⭐
NLP 动态切分 + 重叠 10%~20% = 准确率 +15%
✓
父子文档分块：入库存子块，喂模型时拼父文档
🏆 真实案例
某 AI 团队用"版面分析 + GPT-4o + 动态切分"处理 300 页双栏 PDF，表格还原率 0% → 92%

坑 02 · 向量化

通用 Embedding 失聪

😰 现象
医疗、法律、工业等专业术语检索不到，"胫骨平台骨折"被当成普通骨折
🔍 原因
通用模型（text-embedding-ada-002）没学过领域语料
💥 后果
检索命中率低，尤其对稀有型号、专业黑话无效
✅ 解法：领域微调
用业务数据 + 对比学习微调 Embedding；混合存稠密 + 稀疏向量（BM25）；生产用 int8 量化省 4 倍内存
7 / 11
坑 03 · 检索层

分数乱加 + 重排太慢

😰 现象 1
BM25 10~20 + 向量 0~1 直接加，向量分被淹没，混合检索名存实亡
⚠️ 画面独有警告
"不能简单相加！0-1 vs 0-∞，量纲不同"——讲师 PPT 红色大号字体
😰 现象 2
重排 200ms，并发 100 直接超时，用户看到"转圈"
8 / 11
✅ 坑 03 解决方案

LambdaMART + 策略路由

✓
多路召回：向量抓语义 + BM25 抓术语，两路并行
⭐
LambdaMART 统一打分（LTR 算法）替代拍脑袋权重
✓
粗精分离：向量+BM25 取 Top 30，重排只算 Top 10
⭐
画面独有数据：Top 30 → Top 3，100ms 降级阈值
✓
更精密模型：BGE-Reranker 精排 + 全局泛读/局部精读
9 / 11
坑 04 · 生成层

模型胡编乱造

😰 现象
回答与检索片段矛盾，或引用不存在的来源
🔍 原因
大模型训练形成的"惯性"，或 prompt 没有强制约束
💥 后果
用户信任度暴跌，企业不敢上线生产环境
✅ 解法：强制引用
Prompt 硬约束"只基于文档回答"；每个结论挂载文档 ID+页码+高亮坐标；Faithfulness 监控一致性
10 / 11
坑 05 · 评估

单一指标盲区

😰 现象
用户点"不满意"，但不知道是没搜到还是模型胡扯
✅ 拆解指标
Context Recall（答案是否在检索片段中）+ Faithfulness（是否忠实于原文）
📊 RAGAS 自动化
监控检索命中率、忠实度、引用精确度
🏆 真实案例
某团队用"指标拆解 + RAGAS 自动化"从 60% 跨越到 85% 准确率
