1️⃣ Embedding（嵌入）—— 把文字变成"坐标"

Embedding 是 RAG 的第一步，也是最基础的一步：让电脑能"算"文字之间的相似度。

1

Embedding · 文本转向量

把"猫在沙发上睡觉"变成 [0.12, -0.34, 0.78, ...] 这种数字串
🌟 类比：给每本书贴 768 个"性格标签"（比如"动物 80%、动作 30%、家具 10%"），然后用这一串数字代表这本书。意思相近的书，标签也相近。

Embedding 的本质是"把语义压缩成数字"。相似意思的句子，得到的数字串也相似；反之差别大。这一步让电脑能做"语义算术"：

国王 - 男人 + 女人 ≈ 女王 → 这就是 Embedding 的魔力

💻 代码示例（用 OpenAI）

from openai import OpenAI
client = OpenAI()

text = "肺结节在 CT 影像上表现为高密度影"

response = client.embeddings.create(
    model="text-embedding-3-small",
    input=text
)
vector = response.data[0].embedding
# vector 现在是 1536 维的浮点数数组
# 比如：[0.0123, -0.0456, 0.0789, ...]
print(f"向量长度：{len(vector)}")
print(f"前 5 个值：{vector[:5]}")

2️⃣ 向量数据库 —— 专门存"坐标"的高速仓库

普通数据库用 B 树查"等于"，向量数据库用 ANN 算法查"最像"，完全是两套逻辑。

2

向量数据库

Pinecone / Chroma / Milvus / Weaviate / Qdrant
🌟 类比：普通图书馆按书名/作者/ISBN 查（精确匹配）；向量图书馆按"主题相似度"查（你给一个主题，馆员把最像的 10 本抱出来）。

向量数据库内部用 ANN（Approximate Nearest Neighbor） 算法——不追求 100% 精确的最相似，而是用 99% 的准确率换 100 倍的速度。常用算法有：

  • HNSW（Hierarchical Navigable Small World）：分层图，最主流
  • IVF（Inverted File Index）：倒排索引，Faiss 默认
  • PQ（Product Quantization）：量化压缩，省空间

💻 代码示例（Chroma 本地版）

import chromadb
client = chromadb.PersistentClient(path="./chroma_db")
collection = client.get_or_create_collection("medical_reports")

# 1. 写入（自动 Embedding）
collection.add(
    documents=["患者男 56 岁，CT 示右肺上叶磨玻璃结节 8mm",
               "心脏 CTA 提示左前降支狭窄 70%"],
    metadatas=[{"date": "2026-05-01", "dept": "放射科"},
               {"date": "2026-05-03", "dept": "心内科"}],
    ids=["r001", "r002"]
)

# 2. 检索（按问题找最像的）
results = collection.query(
    query_texts=["肺上有磨玻璃影的病人"],
    n_results=3
)
for doc in results["documents"][0]:
    print(doc)

3️⃣ 文档切片（Chunking）—— 把长面包切成面包片

Embedding 模型有"一次性吃不下"的问题（8K token 上限），所以必须把长文档切成 200-800 字的小段。

3

文档切片 · Chunking

长文档 → 200-800 字的小段，每段独立 Embedding
🌟 类比：一本书 500 页，检索时不能把整本塞给 AI（它会迷糊）。所以要切成 500 段，每段 1 页，标上页码。问问题时，先找到"页码"再取内容。

切片的 3 个核心参数：

  • chunk_size：每段多少字（中文 200-800 / 英文 200-1000 token）
  • overlap：相邻段重叠多少（避免切断句子）
  • 分隔符：按句号？按段落？按 Markdown 标题？

💻 4 种常见切片策略

# 1. 固定大小切片（最简单）
from langchain.text_splitter import CharacterTextSplitter

splitter = CharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
    separator="\n\n"
)
chunks = splitter.split_text(long_text)

# 2. 递归切片（按段落 → 句号 → 逗号，逐级回退）
from langchain.text_splitter import RecursiveCharacterTextSplitter
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500, chunk_overlap=50
)

# 3. Markdown 切片（按标题分级）
from langchain.text_splitter import MarkdownTextSplitter
splitter = MarkdownTextSplitter(chunk_size=500)

# 4. 语义切片（用 Embedding 检测语义断点）
from langchain_experimental.text_splitter import SemanticChunker
splitter = SemanticChunker(embeddings, breakpoint_threshold_type="percentile")

4️⃣ 相似度检索 —— 找"最像"的几段

把问题也转成向量，然后在向量库里算"距离"，找 Top K 个最近的。

4

相似度检索 · Cosine Similarity

两个向量"夹角"越小，越相似
🌟 类比：在图书馆 100 万本书里，每本书有个 768 维的"主题坐标"。你走进来说"我想看人工智能的"，系统算你和你每本书的"主题夹角"，挑出夹角最小的 10 本。

3 种主流相似度算法：

算法公式（白话）特点适用
Cosine 相似度 看两个向量的"方向" 最常用，不受长度影响 文本检索
欧氏距离 L2 看两个点的"实际距离" 受向量长度影响 图像/位置
点积 Dot Product 两向量相乘求和 计算最快 已归一化的向量

💻 代码示例（手算 Cosine 相似度）

import numpy as np
from numpy.linalg import norm

def cosine_similarity(a, b):
    return np.dot(a, b) / (norm(a) * norm(b))

# 例：两个句子
v1 = np.array([0.8, 0.6, 0.1])  # "猫在沙发上睡觉"
v2 = np.array([0.7, 0.5, 0.2])  # "猫咪躺在沙发休息"
v3 = np.array([-0.5, 0.3, 0.8]) # "今天股票涨了"

print(f"猫 vs 猫: {cosine_similarity(v1, v2):.3f}")  # 0.97（相似）
print(f"猫 vs 股票: {cosine_similarity(v1, v3):.3f}") # -0.32（不相似）

5️⃣ Rerank（重排序）—— 粗排后的"精挑细选"

向量检索速度快但"粗"。用 Rerank 再精排一次，准确率大幅提升。

5

Rerank · Cross-Encoder 精排

不只看"像不像"，还要看"是不是真正答这个问题"
🌟 类比：HR 招聘：先让猎头按关键词搜 100 份简历（粗排，速度快）→ 再让部门经理一份份细看，挑出最合适的 5 个（二排，质量高）。

向量检索用的是 Bi-Encoder（问题和文档分开算向量，再算相似度），速度极快但只看"语义匹配度"。

Rerank 用 Cross-Encoder：把"问题+文档"一起塞进模型，让模型判断"这段文字是不是真的在回答这个问题"。

100 段粗排结果 → Cross-Encoder 精排 → Top 5 喂给 LLM

💻 代码示例（Cohere Rerank）

import cohere
co = cohere.Client("YOUR_API_KEY")

docs = [
    "RAG 的核心是检索增强生成。",
    "猫是哺乳动物。",
    "向量数据库用于存储 Embedding。"
]

results = co.rerank(
    query="RAG 的工作原理是什么",
    documents=docs,
    top_n=2,
    model="rerank-multilingual-v3.0"
)

for r in results.results:
    print(f"分数: {r.relevance_score:.3f} - {docs[r.index]}")
# 输出（r1 排前）：
# 分数: 0.95 - RAG 的核心是检索增强生成。
# 分数: 0.41 - 向量数据库用于存储 Embedding。

6️⃣ Query 改写 —— 把"含糊问题"变清晰

用户的问题往往含糊、错字、省略。Query 改写让 AI 先"重述"问题再检索。

6

Query Rewriting · 查询重写

"腰腿疼咋办" → "腰椎间盘突出的非手术治疗方案"
🌟 类比：顾客说"那个东西"，你听不懂；店员会问"您是想找手机？电脑？还是配件？"。AI 改写就是这个"确认需求"的过程。

3 种改写策略：

  • 扩写：把口语变专业术语（"腰腿疼" → "腰椎间盘突出症状"）
  • 多 Query：1 个问题拆成 3 个角度（原始 + 同义词 + 反义）
  • 指代消解：把"它"、"那个"换成具体对象

💻 代码示例（多 Query 拆解）

from langchain.prompts import PromptTemplate

prompt = PromptTemplate(
    input_variables=["question"],
    template="""请把这个问题改写为 3 个不同角度的搜索 query：
原问题：{question}

输出格式（每行一个）：
1. 原始
2. 专业术语版本
3. 具体场景版本
"""
)

# 输入：
question = "腰疼怎么办"

# 输出（LLM 生成）：
# 1. 腰疼怎么办
# 2. 腰椎间盘突出 保守治疗方案
# 3. 久坐导致的腰部酸痛缓解方法

7️⃣ Metadata 过滤 —— 按时间/部门/作者"标签"筛选

相似度检索只能找"像"的，Metadata 过滤能找"对"的（按业务条件精准筛）。

7

Metadata Filter · 元数据过滤

先按"标签"卡范围，再在范围内找"最像"
🌟 类比：档案柜里 100 万份文件。直接找"合同"会找到 10 万份；先说"2026 年的 + 法务部的"只剩 300 份，再从中找最相关的就快得多。

常见 Metadata 字段：

字段示例作用
时间2026-05-01筛"近 30 天" / "去年"
部门法务部 / 销售部按组织架构筛
作者张三 / 李四按人筛
类型合同 / 报告 / 邮件按文件类型筛
权限公开 / 内部 / 机密按访问权限筛

💻 代码示例

results = collection.query(
    query_texts=["肺癌治疗方案"],
    n_results=5,
    where={                        # Metadata 过滤
        "$and": [
            {"department": "肿瘤科"},
            {"date": {"$gte": "2026-01-01"}},
            {"status": "已审核"}
        ]
    }
)
# 范围从 5 万份缩到 200 份，再找最像的 5 段

8️⃣ 混合检索 —— 关键词 + 向量双管齐下

关键词检索找"完全匹配"的，向量检索找"意思相似"的。混着用效果最好。

8

混合检索 · Hybrid Search

关键词（BM25）+ 向量（Cosine）→ 加权融合
🌟 类比：你问"iPhone 16 Pro Max 价格"。关键词检索找包含这串字的页面（精确）；向量检索找"苹果最新旗舰手机价位"的页面（语义）。两个结果合并去重 = 最全。

融合公式（RRF）：

最终分数 = 0.5 × 关键词排名分 + 0.5 × 向量相似度分
# 权重可调：
# - 搜产品型号 / 错误码 → 关键词权重高（0.7）
# - 搜解决方案 / 思路 → 向量权重高（0.7）

💻 代码示例（Qdrant 混合检索）

from qdrant_client import QdrantClient, models

client = QdrantClient("localhost", port=6333)

results = client.search(
    collection_name="docs",
    query_vector=embedding,           # 向量检索
    query_filter=None,
    limit=10,
    # 启用 BM25 关键词混合
    search_params=models.SearchParams(
        hybrid=models.HybridQuery(
            fusion=models.FusionQuery(
                fusion_type=models.Fusion.RRF  # 倒数排名融合
            )
        )
    )
)

🔧 真实案例：Elasticsearch 混合搜索

开发文档搜"401 unauthorized"

Top 3 准确率：95%
错误率：-40%
开发满意度：+35%

9️⃣ Prompt 组装 —— 把"资料 + 问题"打包给 LLM

最后一步：把检索到的段落 + 用户问题 + 角色设定 + 格式要求拼成完整 Prompt 喂给 LLM。

9

Prompt 组装 · Final Prompt Engineering

系统提示 + 检索结果 + 历史对话 + 用户问题
🌟 类比：博学老人（LLM）记不全所有事。你给他递一张卡片："你是一位医生（角色）。参考资料：XXX（检索结果）。患者问：YYY（用户问题）。请基于参考资料回答，不知道就说不知道。"

标准 Prompt 模板（5 部分）：

  1. 角色设定：你是一位资深放射科医生
  2. 参考资料：[1] 段落 1 ... [2] 段落 2 ...
  3. 历史对话：user: ... assistant: ...
  4. 用户问题：当前轮问题
  5. 输出要求：3 句话内、列出参考编号

💻 代码示例

prompt = f"""你是一位资深的放射科医生。
基于以下参考资料回答问题，不知道就说"信息不足"。

参考资料：
[1] {chunk_1}
[2] {chunk_2}
[3] {chunk_3}

历史对话：
{history}

当前问题：{user_question}

回答要求：
- 基于 [1][2][3] 引用
- 不超过 200 字
- 末尾标注参考编号
"""
